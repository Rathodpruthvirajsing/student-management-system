1. Project Profile

1.1 Project Definition

The Student Management System (SMS) is a web-based application designed to automate the administrative tasks of an educational institution. It handles student registrations, course management, attendance tracking, and fee status monitoring through a centralized platform.

1.2 Project Description

The system provides separate portals for Administrators and Students. Admins can manage the core data (students, teachers, courses) and track institutional performance. Students can view their personal profiles, attendance records, exam results, and fee payment history.

2. Requirement Analysis

2.1 Feasibility Study

The project is technically feasible using the LAMP/WAMP stack and operationally feasible due to its intuitive navigation system and role-based access control.

2.2 System Modules

  • Auth Module: Handles secure login and session management.
  • Student Module: Management of student records.
  • Course Module: Course and curriculum management.
  • Attendance Module: Daily presence tracking.
  • Fees Module: Payment logging and balance tracking.

3. Design Phase

3.1 Data Dictionary Snippet

The system uses a relational database schema centered around the student enrollment number.

  • Users Table: Authentication credentials.
  • Students Table: Personal and academic data.
  • Attendance Table: Daily activity logs.

4. Implementation Details

4.1 Robust Path Resolution

To handle XAMPP's environment variables correctly across different directory depths, the following implementation was used:

$project_root = str_replace('\\', '/', dirname(__DIR__));
$script_dir = str_replace('\\', '/', dirname($_SERVER['SCRIPT_FILENAME']));
$relative_path = trim(str_ireplace($project_root, '', $script_dir), '/');
$depth = $relative_path ? substr_count($relative_path, '/') + 1 : 0;
$asset_prefix = str_repeat('../', $depth);
